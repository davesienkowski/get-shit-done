    - args[0] required: workflow name (e.g. 'execute-phase'); args[1] optional: '--phase' flag, args[2] optional: phase number

    - Checks in order (all read-only):
      a) .continue-here.md existence in projectDir root: if exists → blocker {gate: 'continue-here', file: '.continue-here.md', severity: 'hard', anti_patterns: ['continue-here.md present — another session may be in progress']}
      b) STATE.md status field: if contains 'error' or 'failed' → blocker {gate: 'state-error', file: '.planning/STATE.md', severity: 'hard', anti_patterns: ['STATE.md status is error/failed']}
      c) If phaseNum provided, read VERIFICATION.md from phase dir: count lines matching /\|\s*FAIL\s*\|/i → warning (not blocker) per FAIL row found: {gate: 'verification-debt', phase: phaseNum, items: ['FAIL: <line>']}
    - Return: { passed: blockers.length === 0, blockers: [...], warnings: [...] }

    check.verification-status (audit §3.8):
    - args[0] required: phase number
    - Find phase dir via findPhase
    - Read VERIFICATION.md from phase dir
    - If missing: return { status: 'missing', score: null, gaps: [], human_items: [], deferred: [] }
    - Parse VERIFICATION.md:
      - score: count of PASS lines / total lines with PASS or FAIL (as fraction string '3/4' or percentage)
      - gaps: array of strings from rows where status is FAIL or MISSING
      - human_items: rows where type column contains 'human' (case-insensitive)
      - deferred: rows where notes column contains 'deferred' (case-insensitive)
    - Return: { status: 'pass'|'fail'|'partial', score, gaps, human_items, deferred }
    - status 'pass': all rows PASS; 'fail': any FAIL; 'partial': mix with no FAIL but some MISSING

    check.ship-ready (audit §3.9):
    - args[0] required: phase number

    - clean_tree: `git status --porcelain` returns empty string
    - on_feature_branch: current branch is not 'main' and not 'master'
    - current_branch: `git rev-parse --abbrev-ref HEAD`
    - base_branch: check if 'main' exists else 'master'
    - remote_configured: `git remote` returns non-empty
    - gh_available: try execSync('gh --version') — true if exit 0
    - gh_authenticated: try execSync('gh auth status') — true if exit 0
    - blockers: array of strings for each false check that blocks shipping
    - ready: blockers.length === 0 AND verification_passed
    - Return all fields
