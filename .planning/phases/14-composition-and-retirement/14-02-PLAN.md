    **Dry-run implementation (per CONTEXT.md locked decision: "clone state in-memory, run mutation, diff original vs result, return diff without writing"):**

    When dryRun=true and the command is in mutationCommands:
    1. **Clone**: Read the state files that the mutation would affect (STATE.md, ROADMAP.md, config.json, frontmatter of target plan) into in-memory strings/objects. Create a temporary directory with copies of these files.
    2. **Execute**: Call the original handler with the temp directory as projectDir, so mutations write to the temp copy instead of the real project.
    3. **Diff**: Compare the original file contents with the mutated temp copies. For each changed file, produce a before/after diff object.

    - Use `mkdtemp` from `node:fs/promises` for temp directory
    - Copy only `.planning/` subtree files relevant to the command (not entire project)

    - Diff is a simple string comparison per file (not a unified diff format -- just before/after content)
