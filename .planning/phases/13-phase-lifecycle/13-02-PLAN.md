    - updateRoadmapAfterPhaseRemoval: removes phase section from ROADMAP, removes checkbox lines referencing phase, removes table rows, renumbers subsequent integer phases in ROADMAP text (Phase 6->5, 06-01->05-01, etc.)

  <action>
    Add to `sdk/src/query/phase-lifecycle.ts`:

    1. **renameDecimalPhases(phasesDir, baseInt, removedDecimal)**: Port from phase.cjs lines 499-524.
       - Read dirs with readdir(withFileTypes), filter for decimals matching baseInt.N pattern
       - Filter to items with decimal > removedDecimal
       - Sort DESCENDING by oldDecimal (critical — avoids rename conflicts)
       - For each: rename dir, then rename files inside that contain the old phase ID
       - Return { renamedDirs, renamedFiles }
       - Use fs/promises rename (async), NOT fs.renameSync

    2. **renameIntegerPhases(phasesDir, removedInt)**: Port from phase.cjs lines 531-564.
       - Read dirs, match pattern /^(\d+)([A-Z])?(?:\.(\d+))?-(.+)$/i
       - Filter to dirInt > removedInt
       - Sort DESCENDING by oldInt, then by decimal
       - Rename dir and files with zero-padded new number
       - Return { renamedDirs, renamedFiles }

    3. **updateRoadmapAfterPhaseRemoval(projectDir, targetPhase, isDecimal, removedInt)**: Port from phase.cjs lines 569-595.
       - Use readModifyWriteRoadmapMd for atomic writes
       - Remove phase section (header + body until next phase header)
       - Remove checkbox lines referencing the phase
       - Remove table rows referencing the phase
       - If NOT decimal: renumber all subsequent integer phases in ROADMAP text (Phase N->N-1, padded references NN-01->newNN-01, depends-on references)
       - Use descending loop from MAX_PHASE=99 down to removedInt+1 (avoids double-renumber)

    4. **phaseRemove handler**: Port from phase.cjs lines 597-661.
       - Validate targetPhase arg exists
       - Check ROADMAP.md exists
       - Find target directory using readdir + phaseTokenMatches
       - Guard against removing executed work: scan for SUMMARY files, require args[1]==='--force'
       - Delete directory with rm(recursive, force)
       - Call appropriate renaming function (decimal vs integer)
       - Call updateRoadmapAfterPhaseRemoval
       - Update STATE.md: decrement total_phases using acquireStateLock + readFile + stateExtractField + write
       - Return { removed, directory_deleted, renamed_directories, renamed_files, roadmap_updated, state_updated }

    - Use separate regex instances to avoid g flag lastIndex bug
