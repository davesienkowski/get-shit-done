    After registering all handlers, if eventStream is provided, wrap the registry's dispatch to emit events for mutation commands:

    Use a pattern where createRegistry stores a reference and the factory wraps mutation handlers at registration time. Alternatively, extend QueryRegistry with a post-dispatch hook. The simplest approach: after creating the registry, for each MUTATION_COMMANDS entry, replace the registered handler with a wrapper that calls the original handler, then emits an event.

    The wrapper pattern:
    ```typescript
    if (eventStream) {
      for (const cmd of MUTATION_COMMANDS) {
        if (registry.has(cmd)) {
          const original = registry.getHandler(cmd); // may need to add getHandler to QueryRegistry
          registry.register(cmd, async (args, projectDir) => {
            const result = await original(args, projectDir);
            const event = buildMutationEvent(cmd, args, result);
            eventStream.emitEvent(event);
            return result;
          });
        }
      }
    }
    ```

    If `QueryRegistry` doesn't have a `getHandler` method, add one:
    In `sdk/src/query/registry.ts`, add:
    ```typescript
    getHandler(name: string): QueryHandler | undefined {
    ```

    `buildMutationEvent(cmd, args, result)` creates the appropriate event interface based on command prefix:
