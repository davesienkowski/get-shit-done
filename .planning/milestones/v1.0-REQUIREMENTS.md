**Defined:** 2026-04-03
**Core Value:** Systematically improve GSD's workflow quality, power-user configuration, and developer experience by adopting battle-tested patterns from PBR

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| PBR branding/naming | GSD keeps its identity — use `gsd-` prefix, `/gsd:` commands, `GSD >` banners |
| Web dashboard | Multi-milestone effort requiring separate web server, auth, UI framework |
| HTTP hook server | Breaks multi-runtime support; GSD's shell hook system is more portable |
| dev-sync agent | PBR-specific for cross-plugin sync; no use case in GSD |
| Autonomous chaining (20+ steps) | Dangerous without proven stability; implement continue with hard stops first |
| Confidence gate for verification | Non-deterministic; binary pass/fail per must-have is more reliable |
| Speculative planning | Plans invalidated by execution findings; sequential planning is correct |
